# تسک لایو کدینگ - سیستم کامنت تو در تو

## هدف

در این تسک نیاز است که در صفحه اصلی پروژه فعلی، یک سیستم کامنت گذاری تو در تو (Nested Comments) پیاده سازی کنید.

تمرکز اصلی این تسک روی کیفیت کدنویسی، طراحی کامپوننت‌ها و مدیریت State
است.

---

## داده اولیه

کار را با داده زیر شروع کنید:

```js
[
  {
    id: 1,
    text: "سلام! این کامنت اول است.",
    replies: [
      {
        id: 2,
        text: "سلام از کامنت دوم!",
        replies: [
          {
            id: 3,
            text: "هوای تهران آفتابی است.",
            replies: [],
          },
        ],
      },
    ],
  },
  {
    id: 4,
    text: "من برنامه نویسی را دوست دارم.",
    replies: [],
  },
];
```

---

## نیازمندی‌ها

### 1. نمایش کامنت‌ها

- همه کامنت‌ها نمایش داده شوند.
- پاسخ‌ها به صورت تو در تو نمایش داده شوند.
- عمق پاسخ‌ها نامحدود است.

---

### 2. قابلیت Reply

برای هر کامنت یک دکمه **پاسخ** نمایش داده شود.

با کلیک روی آن:

- یک اینپوت برای وارد کردن متن پاسخ نمایش داده شود.
- یک دکمه برای ثبت پاسخ نمایش داده شود و با فشردن آن، پاسخ برای همان کامنت ثبت شود.
- پس از ثبت پاسخ، فرم بسته و اینپوت پاک شود.

---

### 3. ظاهر

روش استایل دهی و استایل نهایی به اختیار شماست، اما نیازی به طراحی پیچیده نیست. کافی است:

- ساختار درختی مشخص باشد.
- پاسخ‌ها با کمی فاصله یا Border از والد قابل تشخیص باشند.
- رابط کاربری خوانا باشد.

در صورت نیاز، TailwindCSS بر روی پروژه نصب و در دسترس است اما استفاده از آن اجباری نمی باشد.

---

## محدودیت‌ها

- از کتابخانه‌های آماده برای نمایش ساختار درختی یا لیست کامنت ها استفاده نکنید.
- مدیریت استیت ها با امکانات خود ریکت انجام شود.
- داده اولیه باید قابل تغییر باشد و بعد از افزودن پاسخ، رابط کاربری
  به‌روزرسانی شود.
